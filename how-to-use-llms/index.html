第 1 章 · 基础

你在和一个
ZIP 文件对话

Karpathy 的心智模型：ChatGPT 是一个 "单标签页的 ZIP 文件" — 互联网的高度压缩快照。它阅读了截至训练截止日期的几乎所有网页、书籍和文档，大约距今 6–12 个月。返回的内容是对这些数据的概率性回忆。

上下文窗口 是它的工作记忆 — 一个有限的 token 磁带，它现在能看到的所有内容。窗口内的内容可以直接访问，窗口外的内容对这次对话而言不存在。除非启用记忆功能，否则会话之间没有持久记忆。

默认情况下没有实时网络连接。模型生成的是提示词最可能的后续内容 — 不是查找，不是搜索，也不是保证。

自我介绍 "你好，我是 ChatGPT。我是一个单标签页的 ZIP 文件。我的知识来自大约 6 个月前阅读的互联网。我只知道这次对话中的内容。我生成的每一个词都是概率性采样 — 请据此对待我。"
上下文窗口 · 实时工作记忆
system
你是一个有帮助的助手。
user
一杯美式咖啡有多少咖啡因？
assistant

user
那双倍浓缩呢？

以上内容对模型可见。一旦窗口填满，旧的上下文就会从边缘滑落并消失。
知识过时提醒 对于咖啡因含量这类永恒事实，模型的权重是可靠的。对于上个月的新闻 — 则不然。在信任答案之前，先了解两者的区别。

第 3 章 · 推理

思考
模型

OpenAI 的 o1、o1 Pro、o3 和 o3-mini 是不同品种 — 所有以 "o" 开头的模型名称都是思考模型。在返回答案之前，它们会运行一段扩展的内部独白：探索方法、回溯、尝试替代方案。

这源自强化学习：模型发现深思熟虑的策略在难题上能带来更好的结果。它尝试不同的想法，回溯，检查推理 — 就像你解决问题时的内心独白。

Karpathy 指出，Claude 3.7 Sonnet（非思考模型）解决了一个 o1 Pro 无法解决的难题。模型选择并不总是显而易见的 — 合适的工具取决于具体任务。

何时使用思考模型 高难度数学、复杂多步代码、形式化推理、逻辑谜题。简单任务不要浪费它们 — 它们更慢、更贵，当没有复杂内容需要推理时，深思熟虑帮助不大。

第 4 章 · 信息

何时
搜索

默认情况下，模型仅依赖其权重运行 — 没有互联网，没有实时数据。启用网页搜索意味着它先检索页面，然后综合答案。这会增加延迟，但解锁了实时信息。

关键问题是：模型过时的回忆是否足够好？ 对于文档完善、永恒的知识 — 是的。对于任何时效性强、近期或小众的内容 — 启用搜索或使用 Perplexity。

  • ✕
    跳过搜索： "一杯美式咖啡有多少咖啡因？" — 文档完善，永恒知识，模型知道
  • ✕
    跳过搜索： "解释瑞利散射" — 教科书物理，无需搜索
  • ✓
    使用搜索： "白莲花度假村第三季什么时候播出？" — 时效性强，发布日期会变动
  • ✓
    使用搜索： "现在去越南旅游安全吗？" — 当前情况可能与训练数据不同
  • ✓
    使用搜索： "最近 USAID 削减是怎么回事？" — 近期新闻，不在训练数据中
  • ✓
    使用搜索： "[某人] 用什么牙膏？" — 小众，可能近期，冷门

第 5 章 · 综合

深度
研究

深度研究 = 扩展思考 + 网页搜索，运行 5–15 分钟。模型并行搜索数十个来源，跨来源推理，生成结构化报告 — 这项工作人类研究员需要数小时。

Karpathy 的演示：研究雷帕霉素与长寿。模型查看了 27+ 个来源，思考了 5 分钟，生成了一份涵盖作用机制（mTOR 抑制）、蠕虫/小鼠/人体试验数据、安全性担忧和正在进行的研究的报告。

ChatGPT 深度研究（需要 $200/月 Pro 版）和 Perplexity 的研究模式都提供此功能。对于文献综述、竞争分析和尽职调查 — 它大幅降低了研究门槛。

最适合 科学文献调研、竞争格局分析、决策尽职调查、医学/法律研究（需验证）。简单事实性问题不值得使用。

第 6 章 · 阅读

文档与
书籍

附加文档将模型转变为阅读助手。上传 PDF、粘贴章节、分享电子表格 — 然后提问、请求摘要，或从内容生成概念图。

Karpathy 的示例：用 Claude 阅读 国富论。"我附上了第一卷第三章 — 请创建这一章的概念图。" Claude 回复 Mermaid 代码，一种图表标记语言，渲染为连接关键概念的关系图。

对于数据，模型可以编写并运行 Python 来生成图表。但把它当作 非常初级的数据分析师 — 擅长写代码，但 Karpathy 发现它在图表中生成了一个虚构的 "1.7 万亿" 数字。始终 scrutinize 数字，而不仅仅是图表形状。

关键工作流 附加文档 → 请求摘要 → 用问答深入各章节 → 请求概念图（Mermaid）→ 对照来源验证任何具体数字。

第 7 章 · 数据分析

代码
执行

ChatGPT 的 高级数据分析 将模型连接到一个实时 Python 运行时。你用自然语言描述任务 — 它编写代码、运行代码、展示结果。无需复制粘贴，无需本地配置。

这是语言与计算的融合。算术、统计、数据清洗、图表生成 — Python 能做的任何事。上传 CSV 请求趋势分析；几秒钟内获得 matplotlib 图表。

Karpathy 的提醒：他发现模型生成的图表中包含一个虚构的 "1.7 万亿" 而非正确数值。代码运行正常；数字是错的。把它当作一个非常有能力但不可靠的初级员工 — 验证数字，而不仅仅是输出形状。

原则 当你需要计算、转换或可视化时使用代码执行。始终检查：生成的代码是否符合你的要求？输出相对于源数据是否看起来合理？

第 8 章 · 开发

智能体
编码

超越聊天，一类新工具将 LLM 直接集成到你的代码编辑器中。Cursor 和 Windsurf 在底层运行 Claude 或 GPT，在你的整个代码库中自主操作 — 读取文件、编写代码、运行命令、迭代。

Cursor 的 Composer (⌘I) 是一个自主智能体循环：描述任务，它就会规划、编写文件、运行 shell 命令、读取错误、循环 — 在任何破坏性操作前请求你的确认。Karpathy 在几分钟内从零搭建了一个 React 应用。

Karpathy 配置中底层的模型：Claude 3.7 Sonnet。关键洞察是，当你足够了解模型以指导和纠正它时，这些工具才最强大，而不仅仅是提示然后祈祷。

第 9 章 · 多模态

语音与
音频

Karpathy 通过 Super Whisper 将大约 一半的查询通过语音路由 — 他在 Super Whisper、WhisperFlow 和 MacWhisper 中选择了前者。按下快捷键，说话，再按一次 — 查询被转录并发送。无需打字，没有摩擦。

ChatGPT 的 高级语音模式 更进一步：音频 token 直接流向和流出自模型，没有文本转录层。结果感觉真正 conversational，不是文字转语音的包装。

NotebookLM (Google) 从你的文档生成音频播客。上传论文、书籍或笔记 — 它生成双人讨论。Karpathy 在散步和长途驾驶时使用它进行被动学习，了解他专业领域之外的话题。

Karpathy 的语音技巧 对于包含产品名称、库名称或技术术语的查询 — 切换到打字。Whisper 经常误转录小众技术词汇。语音最适合自然语言问题。

第 10 章 · 视觉输入

视觉与
摄像头

现代 LLM 接受图像作为输入 — 照片、截图、扫描件、图表。模型对视觉内容的推理与对文本的推理一样流畅，依赖于包含数十亿图像-文本对的训练数据。

Karpathy 的示例：上传 血液检查扫描件 进行解读，将摄像头对准 Aeronet 4 CO2 监测仪 识别设备并解读 713 PPM 读数，展示 指环王地图 它正确识别为中土世界。

视觉对于文档完善的主题最可靠 — 血液检查参考范围、常见消费设备、著名地图 — 训练数据充分覆盖的领域。对于专有或罕见物体，预期更多幻觉。

视觉的强用例 识别未知物体、解读标准实验室结果、解释图表和示意图、印刷文本 OCR、手写识别、分析截图。

第 11 章 · 个性化

记忆与
个性化

默认情况下，每次对话都是无状态的 — 标签页关闭时模型忘记一切。两个功能改变了这一点：记忆（ChatGPT 自动跨会话保存关于你的事实）和 自定义指令（一个持久的系统提示词，塑造每次回复）。

Karpathy 的自定义指令：请求教育性框架（"尽可能教育性地解释"），为语言学习设置韩语敬语等级，分享关于他工作和兴趣的上下文。

将自定义指令视为你的个人系统提示词 — 它在每次对话前加载。好的指令压缩了你否则会在每次查询中重复的首选项，让每个会话感觉它 already 了解你。

自定义指令入门模板 "简洁表达。能用代码就不用文字。当我给你文档时，先用一段话总结。明确标出你的假设。我从事 [你的领域] 工作。"
自定义指令 · ChatGPT
ChatGPT 应该了解你什么？
我是一名对机器学习感兴趣的软件工程师。我喜欢简洁、技术性的回答。我正在学习韩语 — 提供韩语文本时，默认使用敬语体（합쇼체）。
ChatGPT 应该如何回复？
解释概念时要有教育性。把最重要的信息放在前面。多用代码片段。明确标出你做出的任何假设。

核心
要点

01
你在和一个 ZIP 文件对话
模型将互联网压缩成了权重。知识大约过时 6–12 个月，输出是概率性的，上下文窗口之外没有工作记忆。它无法验证自己的答案。
基础
02
了解你的层级和模型
免费 → 有限。$20/月 → GPT-4o / Claude Sonnet。$200/月 → o1 Pro、深度研究。为任务匹配合适的模型 — 思考模型用于难题推理，快速模型用于简单查询。
模型
03
仅对时效性信息使用搜索
对于永恒、文档完善的知识 — 权重足够，跳过搜索。对于近期事件、变动情况或小众话题 — 启用搜索或使用 Perplexity。
搜索
04
深度研究用于多来源综合
5–15 分钟，20–30 个来源，结构化报告。对文献综述和尽职调查真正有用。目前在 ChatGPT 上需 $200/月 付费墙；Perplexity 更便宜。
研究
05
验证代码和数据输出
高级数据分析运行真实的 Python — 但模型可能在编写的代码中虚构数值。对照源数据检查数字，而不仅仅是图表的视觉形状。
代码
06
语音消除了一半的摩擦
基于 Whisper 的听写工具消除了打字障碍。Karpathy 约 50% 的查询通过语音路由。技术产品名称和库名称用文本输入，因为 Whisper 会误转录。
语音
07
ChatGPT 目前是默认选择
功能最多，生态最大，体验最完善。编码用 Claude。搜索优先用 Perplexity。格局变化很快 — 在投入前查看 LM Arena 了解当前排名。
生态
